UI Prototype

This template enables you to quickly generate a single page HTML prototype application that is actually a functioning application and the basis of producing an end product.

UI Prototype uses HTML 5, Bootstrap 3 for CSS styles, JQuery 2 for Javascript.

UI Prototype - Intro

The basis of this HTML template are individual pages that are represented in <section> tags. Each section has an id and is defined as hidden.

<section id="intro" style="display:none">
</section> <!-- intro -->

To render any given page defined as a section you add to a standard bootstrap link or button the "render" class and you define a data-section element defining the section you wish to render. On the home page you clicked a button to get to this intro section. 

<button class="btn btn-success render" data-section="intro">Intro<button>

Behind the scenes there is a jQuery click() function for the defined .render class. This takes care of the appropriate showing of the section. It does a lot more also, we discuss that later.

UI Prototype - Layout

Bootstrap enables a responsive CSS design of your webpage. You can customize this for your application as you wish. The following is the default that is used here and works to provide a view for three responsive layouts (cell phone, tablet, computer). 

<section id="layout" style="display:none">
  <div class="container-fluid">
    <div class="row-fluid row-centered">
      <div class="col-xs-12 col-sm-8 col-md-6 col-sm-offset-2 col-md-offset-3">
        <h1>UI Prototype - Layout</h1>
        <p>...</p>
      <div> <!-- col -->
    <div> <!-- row -->
  <div> <!-- container -->
</section> <!-- layout -->

You can resize this window down to see the responsive design. Code blocks are not the usual application so this becomes less effective in a smaller cell phone size resolution.

Within this HTML template there is a section _template. This can be used to cut/paste your default section settings. It works like any other section, try it.

UI Prototype - Forms

Forms that enable the adding/editing of data is one of the two basic functions any website has (the other is views).

You can start off with any of the standard Bootstrap forms that provide various layout options. However this UI protoype uses a number of conventions for presentation which is totally configuration for your own application. Here is our demo Personal Details form. This form provides no working functionality, we discuss that next.

<form class="form-horizontal" method="POST">
  <fieldset>
    <legend>Personal Details</legend>
    <div class="msg alert" style="display:none"></div>
    <div class="form-group">
      <label for="first_name">First Name</label>
      <input name="first_name" class="form-control" />
    </div> <!-- form-group -->
    <div class="form-group">
      <label for="last_name">Last Name</label>
      <input name="last_name" class="form-control" />
    </div> <!-- form-group -->
    <div class="form-group text-center">
      <button class="btn btn-success save">Save</button>
      <button class="btn btn-default cancel">Cancel</button>
    </div> <!-- form-group -->
  </fieldset>
</form>

In addition to the normal form, input and button sections for a general HTML form we add two characteristics.

  • A fieldset and corresponding legend to identify the form.
  • We add an Bootstrap alert div tag with a specific class of msg. When defined the javascript that is used to validate, save and display information will post messages to this section.

If you want to jazz up your form, as with the actual login form in this demo, you can add a well around the fieldset.
